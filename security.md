# Web 安全

## 认证

- 弱密码检测
- 多因素验证
- 限制或延迟登录尝试，但不要直接拒绝

## 授权

- 按最小权限法严格管理权限
- 即时注销用户权限
- 加密所有外部存储的敏感信息
- 禁用 public 缓存

## 注入攻击

- 编码时默认一切用户输入均不可靠
- 使用参数化 API 自动转义，防止反射注入字符

## 跨站请求伪造(CSRF)

- 严格设置响应首部`Set-Cookie`
- 检查请求首部`Referer`
- 严格设置跨源资源共享(CORS)

## 跨站脚本攻击(XSS)

- 前端严格设置内容安全策略(CSP)
- 前端使用框架自动转义字符，防止注入 HTML 页面
